## Timeout과 장애전파 방지

> 적절한 타임아웃 설정은 예측하지 못한 외부 서비스나 네트워크 장애에 자원을 오래 점유당하지 않도록 해준다.

```java
    public void callExternalApi(ExternalRequest reuqest) {
        webClient.post()
            .uri("/api")
            .contentType(MediaType.APPLICATION_JSON)
            .bodyValue(request)
            .retrieve()
            //명시적으로 5xx Status 예외도 변환
            .onStatus(status -> status.is5xxServerError(), clientResponse ->
                    clientResponse.bodyToMono(String.class)
                        .flatMap(errorBody ->
                            Mono.<Throwable>error(new ExternalServerErrorException("외부 서비스 500에러 발생"))
                        )
            )
            .bodyToMono(ExternalResponse.class)
            .onErrorMap(throwable -> {
                    if (throwable instanceof ReadTimeoutException ||
                            (throwable.getCause() != null && throwable.getCause() instanceof ReadTimeoutException)) {
                        return new ExternalServiceTimeoutException("외부 서비스 타임아웃 발생", throwable);
                    }
                    return throwable;
                })
    }

```

- 5xx Status 의 경우에도 도메인 예외에 맞게 변환하여 다루어야 한다.
- Timeout 장애 역시도 도메인 예외에 맞게 다룬다.
    - 특히 Retryable 한 경우가 많으므로 부모 예외를 잘 설정하여 복구 가능한 시스템을 만든다.
